cancelPromise(config) → {Promise}

- How long to wait before we exit process - why use this ? If the promise never resolves or takes too long, so we can cancel it when `{maxWait}` time expires
Parameters:
Name Type Description
config Object {defer,checkEvery,maxWait,cbErr,message,logging,id}
Properties:
Name Type Description
config.defer Promise (required)` resolved when process complete or called from callback on timeout
config.checkEvery number (required) how frequently to check if promise is resolved
config.maxWait number (required) long to wait before execiting with cbErr
config.cbErr function (required) called on timeout `cbErr(({error,defer,id}))` > here you can either resolve or reject the pending promise
config.message string (optional) defaults: `taken too long to respond` of provide your own
config.logging boolean (optional) when true will pront waiting process
config.id string | number (optional) added to error callback, and to logging when enabled
Source:

Type

    maxWait: 3000, // expire promise 
    message: 'waited too long', // << use this error message
    logging: true, // display process
    id: new Date().getTime(), // custom id to display or on error
    cbErr: function({ error, defer, id }) {
        // update our reject message
